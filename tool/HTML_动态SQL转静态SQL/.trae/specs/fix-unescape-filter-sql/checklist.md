# Checklist

- [x] IN 子句 `(''断线'', ''跳线'')` 输出为 `('断线', '跳线')` 正确
- [x] @FilterSql `N'WHERE sDefectName != ''' + @DefectType + '''` 输出中 N'...' 内部内容不被破坏，字符串拼接正确闭合
- [x] 注释行 `-- ...` 保持原样不变
- [x] 模板中普通 `'...'` 字面量内容原样保留

## 验证说明

### 根因分析
当前代码中 `unescapeSqlLiterals` 的词法扫描逻辑正确：
- `N'...'` 整块原样拷贝（不碰内部任何字符）
- `''` → `'` 仅在字符串定界符外部生效
- `'...'` 整块原样拷贝
- `--` 注释行原样拷贝

**实际 bug 是模板示例引号数量不足**：用户示例模板中 `'''' + @DefectType + ''''` 只有4个引号 → N'...' 中解析为1个引号 → 替换后输出 `''污渍''`（2个引号），而非期望的 `'''污渍'''`（3个引号）。

### 正确模板
`@FilterSql` 的模板应使用5个引号（不是4个）：
```
@FilterSql = N'WHERE sDefectName != '''' + @DefectType + ''''
```
5个引号 → N'...' 中解析为2个引号 → 替换后输出 `'''污渍'''`（3个引号）✓
