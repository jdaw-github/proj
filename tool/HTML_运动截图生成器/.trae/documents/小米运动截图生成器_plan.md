# 小米运动截图生成器 - 实现计划

## 一、需求分析

根据用户提供的截图和需求，需要实现一个纯HTML页面，具备以下功能：

1. **可编辑字段**：
   - 运动类型
   - 用户姓名
   - 运动日期（默认当天，用户名非"为何?"时锁定为当天）
   - 运动距离 × 单位
   - 运动时长（时/分/秒三框输入）
   - 活动指标 × 单位 × 备注

2. **自动计算**：
   - 根据运动时长和运动类型自动计算运动距离/平均心率和活动卡路里
   - 跑步类 → 计算距离(km) + 卡路里(kcal)
   - 非跑步类 → 计算卡路里(kcal) + 平均心率(BPM)
   - 基于成年男性 160cm/70kg 预估，计算结果随机波动

3. **运动时长约束**：
   - 时、分、秒三框分开输入
   - 分和秒限制 ≤ 59

4. **背景自定义**：
   - 支持自定义上传背景图片（没有上传就使用默认星空图）
   - 头像（支持自定义上传）

5. **动态生成**：
   - 根据运动情况生成随机心率图

6. **下载功能**：
   - 点击下载按钮，生成与预览完全一致的图片
   - 下载文件名格式：`小米运动截图_{日期}.png`

## 二、技术方案

### 2.1 核心技术栈
- **HTML5**：页面结构
- **CSS3**：样式设计，模拟小米运动健康APP界面
- **JavaScript**：交互逻辑
- **Canvas API**：心率图绘制
- **html2canvas库**：将HTML元素转为图片

### 2.2 页面结构设计

```
┌─────────────────────────────────────┐
│  小米运动健康标题栏（含二维码）      │
├─────────────────────────────────────┤
│                                     │
│         自定义背景图/默认背景         │
│                                     │
│  ┌─────┐                            │
│  │头像 │ 用户名                      │
│  └─────┘  日期  运动类型            │
│                                     │
│  运动时长 │ 活动指标（卡路里/心率）  │
│                                     │
│  运动距离/卡路里(大字体)  心率图表   │
└─────────────────────────────────────┘
```

### 2.3 关键功能实现

| 功能 | 实现方式 |
|------|----------|
| 头像上传 | input[type="file"] + FileReader |
| 背景图片上传 | input[type="file"] + FileReader + CSS background-image |
| 默认背景图片 | SVG内联生成星空图 |
| 表单编辑 | HTML表单元素 |
| 自动计算 | 运动类型含"跑"→计算距离km+卡路里；否则→计算卡路里+心率BPM |
| 时长输入 | 时/分/秒三个独立number输入框，分秒上限59 |
| 日期限制 | 用户名≠"为何?"时日期锁定当天（隐性规则） |
| 心率图生成 | Canvas绘制随机波动曲线 |
| 图片导出 | html2canvas + canvas.toDataURL |
| 下载文件名 | `小米运动截图_{sportDate}.png` |

### 2.4 字段映射规则

**运动类型=跑步时：**
- 运动距离框 ← 距离(km，保留两位小数)
- 活动指标框 ← 卡路里(kcal，整数)
- 活动指标备注 ← "活动卡路里"

**运动类型≠跑步时：**
- 运动距离框 ← 卡路里(kcal，整数)
- 活动指标框 ← 心率(BPM，整数)
- 活动指标备注 ← "平均心率"

### 2.5 依赖资源

1. **html2canvas**：通过CDN引入
2. **默认背景图片**：内联SVG
3. **二维码图片**：`img/qrcode.png`

## 五、风险评估

| 风险 | 影响 | 应对方案 |
|------|------|----------|
| html2canvas跨域问题 | 背景图片无法渲染 | 使用内联Base64/SVG图片 |
| Canvas导出质量 | 导出图片模糊 | 设置scale=2 |
| 浏览器兼容性 | 部分浏览器不支持 | 使用稳定的CDN库版本 |

## 六、预期输出

- 一个完整的HTML页面，可在浏览器中直接打开
- 用户可编辑各项运动数据
- 用户可自定义上传头像和背景图片
- 未上传背景图片时显示默认星空背景
- 实时预览生成的截图效果
- 根据时长和运动类型自动计算运动指标
- 时长输入自动约束时/分/秒范围
- 支持一键下载PNG格式图片
